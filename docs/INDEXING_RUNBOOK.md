# INDEXING_RUNBOOK

## 1. Назначение

Документ описывает процедуру подготовки данных, присвоения ID, валидации CSV и сборки индекса для MVP.

## 2. Когда использовать runbook

- при первичной загрузке базы;
- при обновлении CSV;
- при добавлении новых фото;
- при изменении словарей;
- при восстановлении индекса после ошибки.

## 3. Входные артефакты

Необходимы:
- актуальный CSV-файл;
- словари нормализации;
- правила присвоения `photo_id`;
- конфигурация путей к `raw`, `normalized`, `index`.

## 4. Выходные артефакты

После успешной индексации должны появиться:
- валидированный dataset;
- файл со stable `photo_id`;
- нормализованные карточки;
- индексированный dataset;
- build manifest с версиями данных и словарей.

## 5. Пошаговая процедура

### Шаг 1. Подготовка исходного CSV
1. Положить файл в `data/raw/`.
2. Проверить кодировку.
3. Проверить наличие всех обязательных колонок.
4. Сохранить hash исходного файла.

### Шаг 2. Синтаксическая валидация
Проверить:
- что все обязательные поля присутствуют;
- `Height` и `Width` — целые > 0;
- `URL файла` не пуст;
- `Какой план?` входит в enum;
- `Формат фото?` входит в enum.

### Шаг 3. Присвоение `photo_id`
Если `photo_id` отсутствует:
- сгенерировать его по порядку;
- формат `PH-000001`.

Если строка уже ранее индексировалась:
- сохранять прежний `photo_id`.

### Шаг 4. Нормализация строковых полей
- lower case;
- trim;
- нормализация `ё -> е`;
- разбиение list-полей по `, `;
- удаление дублей;
- приведение к каноническим значениям по словарям.

### Шаг 5. Обогащение данных
- вычислить `object_id`;
- вычислить `color_group`;
- извлечь `materials`;
- сформировать `search_text`.

### Шаг 6. Проверка media reference
Для каждой карточки:
- проверить `URL файла` или `Google ID файла`;
- если media reference нерабочий, пометить карточку как `data_quality_issue`.

### Шаг 7. Сборка индекса
- собрать индексируемый набор карточек;
- отсортировать записи по `photo_id`;
- сохранить build manifest.

### Шаг 8. Smoke-check
Проверить 10–15 эталонных запросов:
- `серые кухни`
- `Игнатьев`
- `Japandi гостиная`
- `DEPO вертикальные кадры`

### Шаг 9. Публикация индекса
- переключить приложение на новый index version;
- логировать `index.build_completed`.

## 6. Build manifest

Каждая сборка индекса должна иметь manifest, например:

```json
{
  "build_timestamp": "2026-03-30T15:20:00Z",
  "schema_version": "1.0",
  "dictionary_version": "1.0",
  "source_csv_hash": "sha256:...",
  "rows_total": 1900,
  "rows_valid": 1898,
  "rows_invalid": 2
}
```

## 7. Правила стабильности ID

### Нельзя
- пересоздавать `photo_id` при каждом reindex;
- менять `photo_id`, если карточка осталась той же сущностью.

### Можно
- присвоить новый `photo_id` только новой строке;
- переносить mapping между индексными сборками.

## 8. Критерии успешной индексации

Индексация считается успешной, если:
- все обязательные поля провалидированы;
- каждая валидная строка имеет `photo_id`;
- индекс собран без падения;
- build manifest сохранен;
- smoke-check проходит без критических ошибок.

## 9. Обработка ошибок

### Отсутствует обязательная колонка
Индексация останавливается.

### Битые значения enum
Строка помечается как invalid, ошибка логируется.

### Нерабочая ссылка
Строка индексируется, но с флагом `data_quality_issue`, если карточка иначе валидна.

### Конфликт `photo_id`
Индексация останавливается до ручного разбора.

## 10. Частота reindex

### Обязательно
- после обновления словарей;
- после массовой загрузки новых фото;
- после изменения search_text logic.

### Опционально
- после корректировки отдельных metadata-полей.

## 11. Команды для scripts

Рекомендуемые CLI-команды:

```bash
python scripts/validate_csv.py
python scripts/assign_photo_ids.py
python scripts/build_index.py
python scripts/smoke_search.py
```

## 12. Роли и ответственность

### Редактор базы
- обновляет CSV;
- соблюдает регламент заполнения.

### Инженер / PM
- запускает валидацию;
- проверяет build manifest;
- подтверждает публикацию индекса.

### QA
- проверяет эталонные запросы после reindex.

## 13. Чек-лист перед публикацией индекса

- [ ] CSV актуален
- [ ] обязательные колонки на месте
- [ ] `photo_id` уникальны
- [ ] словари актуальны
- [ ] build manifest создан
- [ ] smoke-check пройден
- [ ] ошибки data quality зафиксированы
- [ ] приложение переключено на новый индекс
